# Dream Review Template

Use this template when `/dream` reviews memory rules, file rules, commands, tests,
or other self-iteration ideas for this QQ bot.

## Fixed Direction

- Keep the current architecture: NapCat/OneBot + onebot-group-proxy + cc-connect
  + local groups/users workspaces.
- Prefer deterministic code for repeatable memory work: routing, classification,
  tags, fingerprints, dedupe, search, soft deletes, pending candidates, and tests.
- Use models only for low-frequency review, summaries, rule proposals, candidate
  patches, and test ideas.
- Treat sub-agent review as advisory input, not execution approval.

## Hard Boundaries

- Do not auto-edit production code.
- Do not apply patches generated by the same dream pass.
- Do not touch secrets, env files, tokens, cookies, provider keys, or browser data.
- Do not enable cross-group search by default.
- Do not restart, reload, deploy, or trigger runtime hot updates.
- Do not add always-on vector databases, local LLM daemons, embedding rebuild loops,
  per-message LLM summaries, recursive review loops, or other heavy background jobs.
- Do not switch the project toward official QQ Bot APIs for this architecture.

## Allowed Outputs

The dream pass may output:

- Review conclusion.
- Boundary risks.
- Deterministic rule improvements.
- Candidate patches, written as suggestions only.
- Test suggestions.
- Rejected ideas and why they exceed the architecture or server budget.

## Required Report Shape

1. Review conclusion
2. Boundary risks
3. Deterministic rule suggestions
4. Candidate patches, not applied
5. Test suggestions
6. Not executed
