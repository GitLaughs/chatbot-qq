# QQ 自用日常群机器人优化方案

生成时间：2026-05-23

## 定位

QQ 机器人面向你和朋友的日常群聊，不按公开产品包装。目标是“像熟人一样低打扰、有记忆、会帮忙、能玩”的私人群助手。

## 当前能力

- NapCat / OneBot 入口。
- `onebot-group-proxy.js` 做群路由、私聊路由、@ 路由。
- 支持健康检查 `/healthz` 和 Prometheus `/metrics`。
- 支持选择性监听、关键词触发、成员画像、群文件保存、PDF 解析、画图命令、梦境维护。
- 服务器已有 `cc-connect-qq`、`onebot-group-proxy`、integrity/cleanup/backup 脚本。

## 个性化原则

- 默认安静：普通聊天不抢话。
- 被叫再动：@、回复 bot、明确求助、匹配群触发词才介入。
- 记忆更强：允许为熟人长期维护画像、偏好、常问领域、待办、文件索引和明确说过要记住的信息。
- 隐私边界：自用群可以更主动做个性化，但仍不把 token、账号、私密文件和敏感身份信息发布到 GitHub。
- 朋友语气：短答、自然、少官腔。
- 群内可控：管理员可临时静音、切模式、看状态。

## P0：稳定和安全

- 保持 `ONEBOT_LISTEN_TRIGGER_MODE=selective`。
- 为每个群维护 `trigger_keywords.txt`，把“作业、代码、报错、画图、总结”等作为触发词。
- 定期跑 `get-chatbot-qq-health-report.ps1`，检查服务、端口、备份、完整性。
- 增加发布前 secret audit，禁止 QQ 号、token、NapCat 配置、日志进发布仓库。
- 保留 daily backup 和 restore test，避免群记忆误删。
- 不继续收紧 systemd sandbox、文件权限或网络限制；这类配置容易影响 NapCat、文件保存、画图和画像读取，必须先在真实群测试再上线。

## P1：日常体验

建议命令：

- `/status`：返回服务、模式、队列、画图任务状态。
- `/quiet 30m`：本群静音 30 分钟。
- `/listen selective|all|off`：切监听模式。
- `/files`：列最近保存文件。
- `/画图`：生图。
- `/dream`：整理群记忆和文件。
- `/help`：说明当前群可用命令。

回复策略：

- 群闲聊：默认不回复。
- 求助问题：先给结论，再给步骤。
- 朋友调侃：短句回应，避免长篇。
- 学习/代码/作业：可详细，但先问是否要完整推导。
- 文件/PDF：先确认保存，再给摘要。

## P2：朋友画像和群记忆增强

每个成员画像只保留：

- 常用称呼。
- 稳定偏好。
- 常问领域。
- 明确说“记住”的事实。
- 当前未完成事项。
- 常用表达和互动边界。
- 喜欢的回答长度、语气和例子风格。
- 对学习、代码、娱乐、生活建议的不同偏好。

群记忆只保留：

- 长期群规则。
- 常用链接/文件。
- 共同项目。
- 重要约定。
- 可公开的梗和偏好。

增强动作：

- 保留自动观察：每条群消息追加到当天 `memory/chat-YYYY-MM-DD.jsonl`，供 `/总结今天` 和 `/dream` 整理。
- 保留显式记忆：用户说 `/记住 内容` 时写入群画像和个人画像。
- 定期收敛：`/dream` 将流水聊天压缩成长期画像，避免无限堆日志。
- 个性化回复：回复前参考成员画像，不同朋友可用不同语气、长度和解释方式。
- 文件习惯：常发文件、常问题型、代码/报告偏好写入个人画像。

## P3：发布边界

QQ 仓库不作为公开产品营销，但 GitHub 更新仍只发布脱敏工程内容：

- 源码、脚本、模板、文档、示例配置。
- 不包含真实 QQ 号、群号、token、日志、群文件、成员画像、NapCat 二进制。

自用上线门槛：

- `npm test` 通过。
- Linux deploy 示例可安装。
- health report 可生成。
- 真实群里 `/status`、`/画像`、`/记住`、`/总结今天`、文件/PDF、画图、重启恢复测试通过。
- 发布仓库 secret scan 通过。
